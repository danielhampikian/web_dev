CSS Works by selecting html elements and applying a rule

A CSS rule consists of a declaration where a property and a value for that property are defined

For example this element, a p element, can be targeted by a css rule that selects it, and applies a color property with a red value to it wherever p elements occur

  • For the html p element, we can target this by inline styling by declaring a style="color:red" attribute for that p element
  • Or we could have a style tag in the head section of html where the decleration p {color:red;} was defined
  • Finally we could have all css rules defined in a seperate .css class that is linked with link rel="stylesheet" type="text/css" href="style.css" in the head section of every html file we want those rules to apply to

One way of targeting multiple elements with css is by defining a CSS class

  • For example, you would define a class with '.classeamplename {color: red}' in a .css file to then target all html elements that have a 'class="exampleclassname" attribute
  • If you want to target individual elements rather than multiple html elements, using an id is more appropriate
  • To use an id, you would have a '#exampleid {color: red}' css decleration in a .css file linked to in html that would then select the element in that page that had a "id="exampleid" attribute
    • When a CSS rule is applied, all things being equal the last CSS rule before the element it is selecting is the one that applies
    • This entails that the order of CSS rules when read by a browser is:
      1. inline elements get applied first
      2. then css rules that are defined internally in the html
      3. finally css rules that are linked to an external file in the reverse order that they are listed
    • For inline styling, a priority of 1000 is added to specificity
    • For an id selector, a priority of 100 is added
    • For a class a priority of 10 is added
    • For an element selector a priority of 1 is added
